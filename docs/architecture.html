Filter Data Flow

1
User selects a filter in FilterBar.vue
4 global filters: Time Period, Location (Warehouse), Category, Order Status. Each selection updates a reactive ref inside the useFilters singleton composable.
2
View component watcher fires
Each view calls useFilters().getCurrentFilters() and wraps it in a watch(). When the returned object changes, the view's loadData() function is called automatically.
3
api.js builds the HTTP request
The centralized api.js module constructs URLSearchParams from the filter object and issues an axios GET. Example: GET /api/orders?warehouse=Tokyo&category=sensors&status=shipped&month=2025-01
4
FastAPI handler filters in-memory data
The route handler calls apply_filters(items, warehouse, category, status) then filter_by_month(items, month). Month values can be YYYY-MM or quarter format (Q1-2025) which is expanded to a 3-month tuple via QUARTER_MAP.
5
Pydantic validates and serializes the response
Filtered Python dicts are validated against Pydantic models (e.g. List[Order]) and serialized to JSON automatically by FastAPI.
6
Vue view stores data and re-renders
The response is stored in a reactive ref (e.g. orders.value = data). Computed properties that depend on it recalculate and the template re-renders — no manual DOM updates needed.

API Endpoints

GET /api/inventory Paginated stock list
warehouse category
GET /api/inventory/{item_id} Single inventory item
GET /api/orders Order list with status filtering
warehouse category status month
GET /api/orders/{order_id} Single order detail
GET /api/dashboard/summary Aggregated KPI metrics
warehouse category status month
GET /api/demand Demand forecasts — no filters
GET /api/backlog Backlog items with PO flag — no filters
GET /api/spending/summary Revenue, cost, profit totals
GET /api/spending/monthly Month-by-month revenue vs costs
GET /api/spending/categories Spending breakdown by category
GET /api/spending/transactions Recent transaction history
GET /api/reports/quarterly Computed quarterly stats from orders
GET /api/reports/monthly-trends Month-by-month fulfillment trends
POST /api/tasks Create task (in-memory)
PATCH /api/tasks/{id} Toggle task completion
DELETE /api/tasks/{id} Delete task
POST /api/purchase-orders Create PO from backlog item
GET /api/purchase-orders/{backlog_item_id} Lookup PO for a backlog item
